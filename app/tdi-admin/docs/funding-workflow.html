Funding System Workflow

How TDI manages grant pursuits from signed contract through award.

Last updated: July 22, 2026

How It Works

Signed Grant Contract → Verify Windows → Request Drafts → Review Narratives → Deliver to School → School Submits → Wait for Decision → Award

Bella owns all school contact and manages the pipeline daily. Rae handles strategy approvals, system configuration, and escalations Bella can't resolve. Agents (Vanessa, Amara) research grant sources and draft narratives. Schools provide information and submit applications.

The Landing Page

Open Admin Portal > Funding to see the school-by-school dashboard. Each school shows:

  • School name, contact, email, pipeline value
  • Next Step banner telling you exactly what to do
  • All grants with status badges (Draft ready, Window open, Agent drafting, Not started)
  • Action buttons: Draft Email, Review Drafts, Request Drafts
  • Google Doc links on drafted narratives ("Open Doc")

Step-by-Step Process

1. Send Intro Email

Each new school needs an introduction from Bella. Click Draft Email on the landing page. The system pre-writes the email with the contact's name and school. Edit if needed, then send. The system logs it and removes the task.

2. Verify Grant Windows

Each school starts with 6 standard grants. Before agents can draft, you need to verify which windows are open. On the school's detail page, each unverified grant shows as a card:

"Verify window: Title II-A" with [Open] and [Closed] buttons. Click one. That's it.

  • Walmart Spark Good - Annual cycle, typically Aug-Sep. Mark as Open.
  • NEA Learning & Leadership - Rolling deadlines, always open. Mark as Open.
  • Title II-A, IDEA, Title I, Community Schools - Check with the school's district office. These are federal/state formula grants that the district handles.
If you don't know whether a window is open: Email the school contact and ask. Use the "Request missing info" button on the school's briefing section to generate the email.

3. Request Narrative Drafts

Once a window is verified open, click Request Draft on the landing page. Choose Vanessa or Amara. The agent will:

  1. Research the school's profile and the grant requirements
  2. Write an A+ narrative with specific budget tables, measurable outcomes, and emotional hooks
  3. Create a Google Doc in the "TDI Grant Narratives" Drive folder
  4. Push the Doc link back to the portal

The grant shows "Agent drafting" while in progress.

4. Review and Approve Narratives

When a draft is ready, the landing page shows "Draft ready" with an "Open Doc" link. Click it to read the Google Doc. If it looks good, approve it in the portal. If it needs changes, edit directly in the Google Doc.

Quality standard: Every narrative must have a specific budget table, measurable outcomes, an emotional opening, and TDI positioned as partner (not applicant). If a draft doesn't meet this bar, send it back. See the Grant Writing Standards skill for details.

5. Deliver to School Contact

After approving, email the narrative to the school contact with submission instructions. Use the Draft Email button to generate the email with the Google Doc link attached.

6. School Submits

The school contact submits the application. TDI cannot submit on behalf of schools. Mark the opportunity as "Submitted" in the portal.

7. Wait for Decision

The follow-up engine automatically monitors deadlines and sends reminders. Track the decision date on the opportunity.

8. Award Allocation

If awarded, update the opportunity status and allocate funds to contract line items.

Key Features

NCES Auto-fill

On any school's detail page, expand School Profile, click Edit, then click "Auto-fill from NCES". The system searches the national school database and fills in enrollment, Title I status, FRL%, address, and NCES ID automatically.

Draft Email Modal

Every email action opens a pre-written email with editable Subject and Body fields. Bella can customize before sending. Emails log automatically.

Notes on Action Items

Click "+ Add notes" on any action item to log activity. Nudge sends auto-log with timestamps. Notes persist and show a green "Notes" button when content exists.

Follow-up Engine

Automated reminders and nudges run hourly. Three safety gates:

  • Window gate: Only sends for open-window opportunities
  • Allowlist gate: Only sends to approved email addresses
  • Prerequisite gate: Skips items whose previous steps aren't done

Google Docs Integration

Every narrative produces a Google Doc in the "TDI Grant Narratives" Drive folder. Docs are shareable, editable, and client-ready.
