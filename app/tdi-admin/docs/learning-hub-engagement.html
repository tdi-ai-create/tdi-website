Learning Hub Engagement System

Duolingo-inspired engagement and confirmation-of-learning system. Interleaved content and comprehension checks, implementation tracking, and learner momentum tools.

Version 1.0 | July 2026 | Internal team reference

Why this exists

Most PD platforms (including our current Hub) deliver content passively -- educators watch a video, maybe answer a quiz at the end, and move on. There's no confirmation they understood the material or changed their practice.

The Duolingo insight

Duolingo doesn't teach then quiz. The quiz is the teaching. Every 30 seconds, the learner proves they understood something. We're applying this principle to professional development -- but going further by also tracking whether educators tried it in their classrooms.

The lesson flow

Instead of "watch everything, then answer questions," lessons now interleave content and engagement checks. A teacher should never passively consume for more than 2-3 minutes without interacting.

Content Quick Check Reflection Try It Checkpoint

Five engagement types

TypePurposeLearner experience
Multiple Choice Comprehension check Pick the right answer. Immediate feedback + explanation. No penalty for wrong answers.
True / False Quick comprehension check Binary choice. Immediate feedback + explanation.
Reflection Depth of understanding Free-text response. Private. Min 50 characters. "This is just for you."
Try It Implementation prompt Specific classroom action to try this week. "I'll try this" button. Shows social proof ("12 educators have tried this").
Checkpoint Lesson summary Key takeaways list. "I've got it" button marks lesson complete.

Minimum density per lesson

Every lesson must have at minimum:

  • 2 comprehension checks (multiple choice or true/false)
  • 1 implementation prompt ("Try It")
  • 1 checkpoint at the end

Reflections are encouraged but not required. Julie Lynn (QA) validates density before publish.

How to set up a lesson (admin workflow)

1
Write lesson body content

In the lesson editor, use the Lesson Body Content section. Write HTML with <h2> or <h3> tags to create sections. The system automatically detects sections and shows a count.

2
Add engagement checks

Below the body content, click "Add Engagement Check" and choose a type. Fill in the question text, answer options (for multiple choice), correct answer, and explanation.

3
Position each check

Each question has a "Position in Lesson" dropdown. Select which content section it should appear after (e.g., "After section 1", "After section 2"). Leave it as "end" if it should appear after all content.

4
Check the flow preview

The Lesson Flow Preview bar at the bottom of the builder shows the visual rhythm. Content segments (gradient), checks (gold), reflections (teal), actions (green), and checkpoints (purple) should alternate naturally.

5
Save the lesson

Click "Save Lesson" to persist the body content. Quiz questions save individually when you click their "Save" button. The learner will see the interleaved flow immediately.

Technical reference

Database tables (Learning Hub Supabase)

TableKey fieldsPurpose
hub_lessons content (JSONB) → body_html Lesson body HTML stored inside content JSONB
hub_quiz_questions lesson_id, question_type, question_text, options, correct_answer, explanation, sort_order, content_position All engagement checks. content_position determines placement between content sections.
hub_quiz_responses user_id, question_id, response, is_correct Learner answers. Upserted on (user_id, question_id).

Content sectioning

The system splits body_html into sections at <h2> and <h3> tags, or at explicit <!-- section-break --> comments. Each section gets an index (0-based). Setting content_position = 0 on a question places it after the first section, 1 after the second, etc. null means "render at the end."

Design principles

These are non-negotiable across all engagement features.

No penalties

Wrong answers get explanations, not point deductions. Teachers are professionals, not students being graded.

No artificial friction

Don't lock lessons behind quizzes. Let teachers move forward freely. Make engagement so natural they want to participate.

Respect their time

Checks should take 15-30 seconds. Reflections have a low character minimum (50). Implementation prompts must be specific enough to act on immediately.

Warm tone

"That's it. Nice work." not "Correct! +10 XP". No emojis, no gamification language, no leaderboards. Personal milestones only.

Phased rollout

BUILT

Phase 1b: Inline Interleaving

Content and quiz questions render interleaved based on content_position. Admin UI for creating, editing, and positioning questions. Flow progress indicator. Section detection from HTML headings.

NEXT

Phase 1a: Content Authoring

Populate every lesson with engagement checks. Minimum density: 2 comprehension checks + 1 action step + 1 checkpoint per lesson. Jasmine (Paperclip agent) drafts questions, team reviews. Julie Lynn (QA agent) validates density.

PHASE 2

Implementation Tracking

Follow-up prompts 5-7 days after a "Try It" action step. Surfaces on dashboard or via Resend email. Three response options: "Tried it" (free text), "Not yet" (defer), "Doesn't apply" (dismiss). Implementation timeline on learner dashboard.

Requires: New hub_implementation_prompts table. Resend integration for email nudges.

PHASE 3

Consistency & Momentum

Weekly engagement streaks (track meaningful interaction, not just login). Four-dimension learning pulse on dashboard: lessons completed, checks passed, actions tried, reflections shared.

New Field Notes recognitions:
"Reflective Practitioner" -- 5 reflections submitted
"Action Taker" -- 3 implementation cycles completed
"Consistent Learner" -- 4-week engagement streak

PHASE 4

Adaptive & Social Learning

Spaced repetition: revisit nudges for incorrectly answered questions. Cohort learning: shared reflections among same-period enrollees. Admin metrics: comprehension rate per lesson, implementation rate, time-to-try.
