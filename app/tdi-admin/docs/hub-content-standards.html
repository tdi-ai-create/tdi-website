How Quick Wins, courses, and content are categorized, tagged, and displayed

Last updated: July 19, 2026 | Version 1.0

1. Quick Win Card Design (Option C)

Design Decision (July 2026)

Quick Win cards use a color-dot design with no thumbnail images. Each category has a unique color. The card shows: category dot + name, title, description (light gray), lift pill, time estimate, and action button. Cards have shadow at rest and lift on hover.

This eliminates the thumbnail bottleneck -- new Quick Wins can publish instantly without waiting for design work. Courses still use photo thumbnails.

2. Quick Win Tagging Requirements

Database enforcement: A trigger (enforce_quick_win_tags_before_publish) prevents any Quick Win from being published with missing required fields. The publish will fail with a clear error message telling you which field is missing.

3. Lift Rating System

Lift = how action-ready a resource is. This determines the colored pill on the card.

ValueLabelPill ColorDefinitionDecision Test
LOWGrab & GoGreenOpen it, use it, done. Zero prep.Can an educator use this by 8 AM if they download it at 7:45?
MEDShort PrepAmberNeeds a planning period to think, customize, then implement.Does this require sitting down to think before using?
HIGHDeep DiveCoralMultiple sessions, deeper reflection, sustained effort.Does this require work across multiple days?

If uncertain, set lift_uncertain = true and assign your best guess. This flags it for human review.

4. PD Hours Formula

PD Hours = 2x Course Duration

A 45-minute course earns 1.5 PD hours. This accounts for implementation time -- the real learning happens when educators apply strategies in their classroom, not just when they watch the content. Rounded to nearest 0.25 hours.

Course DurationPD Hours EarnedExample
5-6 min0.25 hrsQuick Plays
20-24 min0.75 hrsDeep Dives
45 min1.5 hrsEscape Rooms

Certificates auto-generate with a unique verification code (TDI-XXXXXXXX) when all lessons in a course are completed.

5. Quick Wins Page Behavior

  • Default sort: Interleaved by category (round-robin) so the page shows variety, not a wall of one category
  • Pagination: Shows 18 items at a time with a "Show More" button. Resets when filters change.
  • Filters: Category tabs, Lift filter (All/Low/Medium/High), Role filter, Danielson domain filter, Saved/favorites
  • Card hover: Lifts 3px with deeper shadow on hover
  • Background: Cool blue-gray (#F5F7FA), not warm beige

6. Course Standards

  • Courses still use photo thumbnails (unlike Quick Wins which use color dots)
  • Lesson completion: educator marks complete via button, or passes engagement check
  • Course completion = all lessons complete. Auto-generates certificate.
  • PD hours stored in hub_courses.pd_hours (2x formula applied)
