Creator Studio Feedback Portal

Standard Operating Procedure -- How feedback flows from creator submission to approved response

Last updated: July 17, 2026 | TEA-33

Overview

The Creator Studio Feedback Portal replaces email as the primary way creators submit work and receive feedback. Every piece of feedback goes through a human approval gate before a creator sees it.

Creator Submits → Anne Marie Drafts Feedback → Bella Reviews → Creator Sees Feedback → Revise or Approve
Golden Rule: Nothing reaches a creator without a human clicking "Approve." Anne Marie drafts. Bella (or Rae) decides what ships.

Status Reference

Every milestone submission moves through these statuses. The review_status field on each milestone tracks this.

StatusWhat It MeansWho Sees ItNext Action
submitted Creator submitted a deliverable. Waiting for review. Admin portal, Anne Marie via find_work Anne Marie drafts feedback (auto) or Bella writes direct feedback
under_review Anne Marie has drafted feedback. Waiting for Bella's approval. Admin portal (Feedback Review Queue) Bella approves, edits, or rejects the draft
feedback_ready Feedback approved and visible to creator. Creator portal + Admin portal Creator reads feedback, revises, or marks complete
revised Creator resubmitted after receiving feedback. Admin portal, Anne Marie via find_work New review cycle begins
approved Deliverable accepted. Milestone can be marked complete. Everyone Milestone advances to next phase

Step-by-Step: Creator Submits Work

1 Creator logs into Creator Portal

URL: teachersdeserveit.com/creator-portal/dashboard

They see the "Submissions & Feedback" section on their dashboard.

2 Creator clicks a milestone under "Ready to Submit"

Only milestones with status available or in_progress appear here.

A submission form opens with two fields:

  • Deliverable (required) -- link to their work, text description, or file reference
  • Notes (optional) -- context or questions for the review team
3 Creator clicks Submit for Review

What happens when they click:

  • A new row is created in creator_milestone_feedback with their submission
  • The milestone status changes to waiting_approval
  • The milestone review_status changes to submitted
  • Submission version auto-increments (1, 2, 3...)
  • Creator's updated_at timestamp bumps (cancels any active re-engagement drip)

The milestone now appears under "Under Review" in their portal with the message: "Submitted -- your team is reviewing this."

Step-by-Step: Anne Marie Reviews (Automated)

4 Anne Marie discovers the submission via find_work

Anne Marie calls the Sync API periodically. When find_work returns a submission_review work item, she picks it up.

She sees: creator name, milestone, what was submitted, submission version, and any notes.

5 Anne Marie drafts feedback via draft_feedback

What happens:

  • Feedback is saved with feedback_draft_status = 'pending_review'
  • Milestone review_status changes to under_review
  • visible_to_creator remains false -- creator cannot see this yet
  • Duplicate guard: only one pending draft per milestone at a time
Note: Anne Marie's skill file needs to be updated to automatically review submissions. Until then, submissions sit in "submitted" status and Bella needs to write direct feedback or wait for Anne Marie to be configured.

Step-by-Step: Bella Reviews Feedback

6 Bella opens the Feedback Review Queue

Location: Admin Portal > Creator Studio > Dashboard tab

The "Feedback Review Queue" section shows all feedback with status pending_review.

Each card shows: creator name, milestone name, what was submitted (truncated), and Anne Marie's drafted feedback.

A count badge shows how many items are waiting.

7 Bella chooses an action on each feedback item

Three options:

Approve -- Feedback is good as-is

  • Feedback becomes visible to the creator immediately
  • Creator receives an email: "You have new feedback on your submission!"
  • Milestone review_status changes to feedback_ready

Edit & Approve -- Feedback needs tweaking

  • An inline text editor opens with Anne Marie's draft pre-filled
  • Bella edits the feedback to her liking
  • Clicks Save & Approve
  • Same result as Approve, but with Bella's edited version

Reject -- Feedback misses the mark

  • Draft is discarded (creator never sees it)
  • Milestone review_status resets to submitted
  • Anne Marie can try again on the next find_work cycle, or Bella can write direct feedback instead
7b Alternative: Bella writes direct feedback (bypass Anne Marie)

Location: Admin Portal > Creator Studio > [Creator Detail] > Submissions & Feedback

Bella can select a milestone from the dropdown, type feedback directly, and click Send Feedback.

This skips the AI draft entirely -- feedback is immediately approved and visible to the creator. Use this when:

  • Anne Marie hasn't drafted anything yet and the creator is waiting
  • The feedback is simple enough that AI review isn't needed
  • Anne Marie's draft was rejected and you want to write it yourself

Step-by-Step: Creator Reads Feedback & Responds

8 Creator receives email notification

Subject: "Creator Studio | New feedback on your submission!"

CTA button links to teachersdeserveit.com/creator-portal/dashboard

9 Creator reads feedback in the portal

Under "Feedback Received," they see a card with:

  • Green status badge: "Feedback Ready"
  • The feedback text from their team
  • Date the feedback was approved
  • Expandable section showing what they originally submitted
10 Creator takes action

Two options:

Revise & Resubmit

  • Opens the submission form pre-filled with their previous deliverable
  • They update their work and resubmit
  • Submission version increments (v2, v3, etc.)
  • The review cycle starts again from Step 4

Request a Call

  • If the feedback needs discussion, creator clicks this button
  • A "Call Requested" flag appears on the admin side
  • Bella or Rae schedules a call at their discretion
  • This is optional -- most feedback should be resolvable in writing

Bella's Daily Routine

Daily check (2 minutes): Open Creator Studio Dashboard. Look at the Feedback Review Queue count. If there are items waiting, review and approve/edit/reject each one. That's it.
  1. Open Admin Portal > Creator Studio > Dashboard
  2. Check the Feedback Review Queue count badge
  3. For each item: Read Anne Marie's draft. If it's good, click Approve. If it needs edits, click Edit & Approve and adjust. If it's wrong, click Reject.
  4. Check Today's Priorities for any call requests (blue badge on submission cards)
  5. Done. Creators get notified automatically.

Rules & Guardrails

Rule 1: No feedback reaches creators without human approval.
Anne Marie drafts. Bella or Rae approves. The system enforces this -- visible_to_creator stays false until someone clicks Approve.
Rule 2: One pending draft per milestone.
Anne Marie can't pile on. If she already has a pending draft for a milestone, the system blocks a second one. Reject the first to let her try again.
Rule 3: Reject resets, it doesn't delete.
When you reject a draft, the milestone goes back to "submitted" status. Anne Marie can draft again, or you can write direct feedback. The creator never knows a draft was rejected.
Tip: Use "Edit & Approve" more than "Reject." It's faster to tweak Anne Marie's draft than to reject and start over. Only reject if the draft completely misses the point.

FAQ

What if a creator submits but Anne Marie doesn't draft feedback?

This can happen if Anne Marie's skill file hasn't been updated or if she's offline. The submission will sit in "submitted" status. Bella can write direct feedback from the creator detail page at any time -- no need to wait for Anne Marie.

What if a creator requests a call?

The "Call Requested" badge appears on the submission card in the admin portal. Bella or Rae schedules a call at their discretion. There's a call_notes field in the database to log what was discussed.

Can a creator resubmit without waiting for feedback?

Yes. The "Revise & Resubmit" button appears on any milestone in "Under Review" status. The new submission creates a new version and restarts the review cycle.

What if I want to mark a milestone as complete without going through the feedback loop?

Use the existing milestone approval flow on the creator detail page. The feedback portal is for milestones that need iterative review (outlines, drafts, recordings). Simple milestones (signed agreement, profile photo) can still be approved directly.

Does the creator get notified for every status change?

No. The creator only gets an email when feedback is approved (made visible). They don't get notified for submitted, under_review, or rejected statuses. They see "Under Review" in their portal, which updates in real time.
