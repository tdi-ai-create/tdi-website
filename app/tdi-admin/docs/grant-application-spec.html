Grant Application Package Standards

Every application TDI prepares must be a white-glove, elite-level deliverable. This spec defines what that looks like.

Last updated: July 22, 2026

Core principle: Our client should be able to open this document, follow the steps, copy and paste the text, and submit a winning application without ever writing a single word or making a single decision. If they have to think, we failed.

Every Application Package Must Contain

1. Welcome Section

  • Personal greeting using the contact's first name
  • One sentence explaining what this document is
  • Numbered table of contents showing what's inside
  • Clear statement: "You do not need to write anything. Everything is ready to copy and paste."
  • Help contact (Bella's email, offer to do a call walkthrough)

2. Step-by-Step Application Instructions

  • Numbered steps, not paragraphs
  • Exact URLs to click (not "go to the website")
  • Every account credential they'll need (NCES number, EIN, etc.) listed upfront so they don't have to search
  • What to do if they get stuck at each step (specific fallback instructions)
  • Expected timeline: "This takes about 15-20 minutes"

3. Pre-Written Application Sections

  • Each section labeled with the EXACT field name from the application form (not our guess)
  • Character counts shown if the form has limits (e.g., "487 of 500 characters")
  • Copy-paste instruction above each section: "Copy this text and paste it into the [Field Name] field"
  • No section should require the client to edit, rewrite, or add information

4. Budget Table

  • Line items with verified prices (not estimates)
  • Formatted to match the application's budget format
  • Total must equal the requested amount exactly
  • Each line item connected to the project description

5. Submission Checklist

  • Printable checkbox list the client can work through
  • Each step references a page/section in the document
  • "Forward confirmation email to bella@teachersdeserveit.com" as the last step

6. Help and Support

  • Bella's email and phone
  • Rae's email as escalation
  • Offer to schedule a call to walk through submission together
  • "We respond within 24 hours" commitment

Writing Standards

Before Writing: Research the Actual Form

Never guess at form fields. Before drafting any application, someone must open the actual application form and document every field name, character limit, and required attachment. If the form says "Tell us about your project" and we label our section "Project Description," the client has to figure out the mapping. That's not white-glove.

ReapplicationCannot reapply to same facility in same fiscal year

The 5 form sections:

  1. Organization Overview (500 chars) — Mission, history, service area
  2. Community Need (500 chars) — Local data, specific problem
  3. Proposed Use of Funds (500 chars) — Budget breakdown with percentages
  4. Expected Outcomes (500 chars) — Quantified results with timeframes
  5. Local Connection (350 chars) — Link to the specific Walmart facility's community
Key insight: 500 characters is about 3-4 sentences. These are SHORT. Every word must count. Do not write paragraphs and then try to cut them down. Write to the limit from the start.

NEA Learning & Leadership ($2,000 to $5,000)

DetailValue
Application deadlineThree cycles per year (check neafoundation.org for dates)
Form sectionsActivities, Need Assessment, Sustainability & Sharing, Budget
Applicant requirementMust be current NEA member (teacher, ESP, or SISP)
Budget formatUpload Word, Excel, or PDF with line items
Critical ruleProject must NOT start before grant notification date. Auto-rejected if it does.

The form sections:

  1. Lead Applicant — NEA member info (name, member number, school, contact)
  2. Activities — How, when, and where will the learning be accomplished?
  3. Need Assessment — Student need AND professional need by the applicant
  4. Sustainability & Sharing — How will this continue beyond the grant? How will knowledge be shared?
  5. Budget — Line items, all directly related to proposed work, cost-efficient
Critical: NEA applications are written in FIRST PERSON from the teacher's perspective, not from TDI's perspective. "I teach at Allenwood Elementary..." not "Allenwood Elementary partners with TDI..."

Quality Checklist (Before Sending to Client)

Every application package must pass ALL of these before delivery:
  • Welcome section with personal greeting and table of contents
  • Step-by-step instructions with exact URLs and credentials listed upfront
  • Every narrative section labeled with the EXACT form field name
  • Character counts shown for each section (if applicable)
  • "Copy and paste" instruction above every narrative section
  • Budget with verified vendor prices (not estimates)
  • Submission checklist with checkboxes
  • Help contact section with Bella's email and call offer
  • Document title starts with "READY TO SUBMIT"
  • Shared with rae@ and bella@ as editors
  • Proofread for dashes (never use --), jargon, and passive voice
  • Local data used, not national statistics
  • All character limits respected (shown in document)
