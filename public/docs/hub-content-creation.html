Hub Content Creation Workflow

How new Quick Wins go from idea to live on the Learning Hub

Last updated: July 21, 2026

The Pipeline

Rae requests→ Izzy drafts→ Julie Lynn QA→ Rae approves→ [BUILD] Publish→ Community seeded→ Nora verifies

Hub content does NOT go through Kristin/CMO. Only social media posts about Hub content need CMO review. If Kristin is paused, Hub content creation is unaffected.

QA Checklist (Step 3)

  • Title is clear and specific
  • Description is 1-2 sentences
  • Category is one of the 12 approved categories
  • Lift rating assigned (LOW/MED/HIGH)
  • Roles assigned (teacher/para/leader/coach)
  • Danielson domains assigned
  • Topic tags assigned (1-5 from approved list)
  • Access tier set (default: professional)
  • Content is actionable, not just theory
  • PDF is formatted and downloadable

Database enforces this: The trigger enforce_quick_win_tags_before_publish will reject any publish attempt with missing required fields.
